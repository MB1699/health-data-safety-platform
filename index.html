Architecture

Five stages, one local stack

01

Ingest

Python generates intentionally messy patient, claim, encounter, and lab feeds to simulate real healthcare fragmentation.

02

Clean

Pandas standardizes names, DOBs, phones, ZIP codes, gender values, and other risky free-text fields.

03

Deduplicate

Explainable matching rules merge identities using MRN, email plus DOB, and name plus DOB with ZIP or phone.

04

Model

DuckDB creates a zero-cost warehouse with a patient 360 mart, data quality views, and run monitoring tables.

05

Monitor

Streamlit exposes pipeline health, duplicate clusters, audit events, and fact-table volumes for operators.

Compliance-friendly audit trail

Every run records stage, status, timestamp, and payload so you can prove what happened without exposing raw PHI in the log.

Simple orchestration, zero cloud spend

Prefect-compatible decorators keep the demo local while still showing how the workflow would be scheduled in production.

@flow(name="healthcare-data-safety-platform")
def healthcare_data_safety_flow():
    raw_counts = generate_task()
